# Seed term coverage e2e test

## Context

We added `tokio` to the seed dictionary and a `::` qualified path guard, but have no
dedicated e2e test that exercises **all** seed terms across the three states:
1. **Bare** тАФ plain text, should be linkified to `` [`Term`] ``
2. **Backticked only** тАФ `` `Term` ``, should upgrade to `` [`Term`] ``
3. **Already linked** тАФ `` [`Term`] `` with ref def, should not be touched

This test acts as a coverage net: if a new term is added to the seed or the linker logic
changes, this test catches regressions across all terms and states.

## Files to create/modify

### 1. New: `test_data/complete_file/input/sample_seed_term_coverage.rs`

Synthetic `.rs` file with `//!` doc comments. Every seed term appears in all three states.
Structure:

- **Section 1: All terms bare** тАФ every term from the seed appears once as plain text in
  natural prose. Grouped by tier (internal, spec, external).
- **Section 2: All terms backticked only** тАФ every term wrapped in single backticks
  (`` `CSI` ``, `` `tokio` ``, etc.). Should upgrade to linked form.
- **Section 3: All terms already linked** тАФ every term as `` [`Term`] `` with matching ref
  defs at the bottom. Should not be modified.
- **Section 4: Edge cases**
  - Qualified paths: `tokio::io::stdin()`, `vte::Parser` тАФ must NOT be split
  - Single colon after term: `ESC:` тАФ must still be linkified
  - Compound spec terms: `` `CSI` spec ``, `` `VT-100` spec `` тАФ must be recognized

All 36 seed terms covered:
- Tier 1 (5): CSI, SGR, ESC, DSR, OSC
- Tier 1b (6): `` `CSI` spec ``, `` `SGR` spec ``, `` `ESC` spec ``, `` `DSR` spec ``,
  `` `OSC` spec ``, `` `VT-100` spec ``
- Tier 2 (25): ANSI, ASCII, UTF-8, DCS, VT-100, VT-220, DECSTBM, DEC, xterm, Alacritty,
  Kitty, RXVT, rxvt-unicode, urxvt, ConPTY, gnome-terminal, ReGIS, Sixel, X10,
  ITU-T Rec. T.416, tokio, vte

### 2. New: `test_data/complete_file/expected_output/sample_seed_term_coverage.rs`

Generated by `regen_golden_output_snapshot_files_for_input_files`.

### 3. Modify: `validation_tests/complete_file_tests.rs`

Add two test functions following existing patterns:

- `test_seed_term_coverage` тАФ golden file comparison + specific assertions:
  - Every bare term is linked
  - Backticked terms upgraded
  - Already-linked terms + ref defs preserved
  - Qualified paths not split
  - Single colon still linkified
  - All expected ref defs present
- `test_seed_term_coverage_idempotent` тАФ 3-run idempotency test

Add `"sample_seed_term_coverage.rs"` to the `regen_golden_output_snapshot_files_for_input_files` files list.

### 4. Simplify: Make `regen_golden_output_snapshot_files_for_input_files` auto-discover all input files

Instead of maintaining a hardcoded list, `regen_golden_output_snapshot_files_for_input_files` should glob all `*.rs` files in
`test_data/complete_file/input/` and regenerate all of them. This way adding a new fixture
never requires remembering to update the list.
