# Documentation Build & Formatting

Invoke the `write-documentation` skill for a **full documentation check**.

This covers all four parts:
1. Structure (inverted pyramid)
2. Intra-doc links (crate:: paths, reference-style)
3. Constant conventions (binary/byte literal/decimal)
4. Formatting (cargo rustdoc-fmt, cargo doc)

Use `/fix-intradoc-links`, `/fix-comments`, or `/fix-md-tables` for targeted fixes.
